In the first place, this is the dashboard page.



Welcome to the Axentra Dashboard — your centralized control panel built with Node.js, Express, and pure HTML/JS/CSS. This interface is crafted for speed, clarity, and a modern neon aesthetic; it gives you a single place to manage bots, view live stats, and access support tools without noise or clutter.

This website is powered by Node.js on the backend and uses standard HTML, vanilla JavaScript, and CSS for the frontend. The server serves static assets and routes cleanly so each page (Dashboard, About, Bot Info, Contact) loads fast and reliably. The architecture keeps logic separated: Node/Express handles routing and static delivery, while the front end focuses on UI, animations, and user interactions.

Features you'll find here:
• Real-time-feel dashboard layout for quick access to bot controls and analytics.
• Dedicated Bot Info page with name, prefix, command categories, and rough server/user stats.
• Unified navigation and a responsive sidebar so everything works smoothly on phones, tablets, and desktops.
• Contact form with client-side validation and friendly UX that’s ready to connect to a backend webhook or API.
• A themed 404 page so misdirected links remain graceful and helpful.

Design & UX notes:
• The interface uses a glassy, neon container to keep content readable over an animated background without sacrificing performance.
• Accessibility and responsiveness were prioritized: scalable type, reachable controls, and a collapsible menu to make mobile navigation frictionless.
• Visual feedback is provided for key actions (button hover/glow, form submission status) so users always know what happened after they interact.

Maintenance & extensibility:
• To add new pages or bot endpoints, create a new route and corresponding static HTML file; the static server will serve it automatically.
• The contact form can be wired to any webhook or REST endpoint for instant notifications (Discord webhook, email API, or custom backend).
• The CSS is modular and centralized — tweak variables, add components, or swap the theme without touching server logic.

Security & performance tips:
• Always sanitize and validate any data sent from the frontend before saving or forwarding it on the server.
• Serve the site over HTTPS in production and set appropriate HTTP headers (CSP, HSTS) to harden the app.
• Use a process manager (pm2, systemd) for Node and enable gzip compression and caching headers for static assets to minimize latency.
